1.Git简介

在实际开发中，Git和Github是合作和迭代的必要工具，所以本文结合流程图对常用的git指令做了一些必要的终结。

git的通过操作流程：

图中的名词详解：

  • workspace（工作区）：本地存放项目文件的地方
  • Index（暂存区）：使用git管理项目文件的时候，我们会发现本地的项目文件会多出一个.git的隐藏文件夹，这个文件夹称之为版本库。其中文件夹中包含了两个部分，一个是暂存区，顾名思义就是暂时存放文件的地方，通常我们使用 git add 的方法将工作区的文件添加到暂存区里
  • Repository（本地仓库）：.git文件夹里还包括git自动创建的master分支，并且将HEAD指针指向master分支。使用commit命令可以讲暂存区中的文件添加到本地仓库中
  • Romote（远程仓库）：不在本地里，而是将项目代码部署在远程git服务器上，比如我们熟悉的github，就是一个远程仓库，通常使用clone命令将远程仓库拷贝到本地仓库中，开发后推送到远程仓库中即可

其中在本地的操作流程可以用一张更加详细的图来说明：

日常开发是我们的代码放置在工作区中，通过add命令将代码提交到暂存区，然后使用commit命令将暂存区提交到master分支，然后在使用push命令提交到远程仓库

通过以上的说明我们可以讲git的命令分为以下几类：

  1. git初始化和管理配置
  2. 工作区和暂存区的交互
  3. 暂存区与本地仓库的交互
  4. 本地仓库与远程仓库的交互

2.Git配置命令

2.1 配置用户信息

  1. 配置用户名： ==git config –global user.name “your name”==
  2. 配置用户邮箱： ==git config –global user.email “youremail@github.com“==

2.2查询配置信息

  1. 列出当前配置： ==git config –list==
  2. 列出repository配置： ==git config –local –list==
  3. 列出全局配置： ==git config –global –list==
  4. 列出系统配置： ==git config –system –list==

3.工作区操作命令

3.1 新建仓库

  1. 将工作区中的项目使用git进行管理，即新建一个新的本地仓库： ==git init==
  2. 从远程git仓库复制项目: ==git clone ==

3.2 提交工作区文件

  1. 提交工作区所有文件： ==git add .==
  2. 提交工作区指定文件： ==git add ==
  3. 提交工作区某个文件夹： ==git add [dir]==

3.3撤销提交

  1. 同时删除工作区文件和暂存区删除对应文件： ==git rm==
  2. 从暂存区里删除文件，而不影响工作区： ==git rm –cached==
  3. 取消暂存区已经暂存的文件： ==git reset HEAD ==
  4. 撤销上一次对文件的操作: ==git checkout –==。当本地文件崩盘但是还没提交，我们可以使用 ==git checkout .==将所有的文件都恢复到上一次add的时候

3.4 更新文件

重命名文件，并将已改名的文件提交到暂存区： ==git mv [file-original] [file-renamed]==

3.5 查看信息

  1. 查询当前工作区所有文件的状态： ==git status==
  2. 比较工作区中当前文件和暂存区的差异，就是修改后还没有使用add命令提交的内容： ==git diff==

4. 暂存区上的操作命令

4.1 提交文件到版本库

  1. 将暂存区的文件提交到本地仓库中，并附加上提交信息： ==git commit -m “commit_info”==
  2. 将所有的文件跳过暂存区直接提交到本地仓库：
    ==git commit -a -m “commit_info”==
  3. 提交文件是，发现漏掉的文件或者注释错误，可以撤销上一次提交: ==git commit -amend==

4.2 分支管理

  1. 创建分支： ==git branch ==
  2. 切换到新分支： ==git checkout ==
  3. 新建并切换到分支： ==git checkout -b ==
    4.删除分支： ==git branch -d ==
  4. 将当前分支和指定分支合并： ==git merge ==
  5. 显示本地仓库的所有分支： ==git branch==
  6. 查看各个分支最后一个提交对象的信息： ==git branch -v==
  7. 将远程分支合并到当前分支： ==git merge /==

5.本地仓库上的操作

  1. 查看本地仓库关联的远程仓库： ==git remote==,在clone完远程仓库后，远程仓库默认为origin，加上-v会显示远程仓库的url地址
  2. 添加远程仓库，一般会去一个简短的别名： ==git remote add [remote-name] [url]==,比如 ==git remote add example git://github.com/example/example.git==
  3. 从远程仓库中抓取本地仓库中没有的更新： ==git fetch [remote-name]==,但是这个不会自动合并工作分支，也可以使用git pull来拉去远程仓库中的数据并和当前分支进行合并
  4. 将本地仓库分支推送到远程仓库上： ==git push [remote-name] [branch-name]==
