1.计算机网络体系结构

1.1定义

计算机网络体系结构实际上就是计算机网络的各层级+其对应的网络协议

1.2结构
  • OSI体系结构：结构清晰但使用范围较窄
  • TCP/IP体系结构：包含了常用的各类网络协议，是Internet的核心协议，广泛应用于局域网和广域网通讯
  • 五层体系结构：融合了以上两者的体系结构，主要目的是用于developer学习网络基础
  • 其中低三层用于网络数据传输，高三层用于计算机系统，主要为数据的处理，传输层用于将网络数据传递到本地。

2.TCP协议

2.1定义

Transmission Control Protocol，即 传输控制协议

  1. 属于传输层网络协议
  2. 基于TCP协议的应用层协议有HTTP,POP3,SMTP,FTP和Telent

其详细介绍可见 大话TCP协议

3.UDP协议

3.1定义

User Datagram Protocol，即 用户数据报协议

  1. 属于传输层网络协议
  2. 基于UDP协议的应用层协议有DNS，SNMP和TFTP
3.2特点

无连接的，不可靠，面向报文，无阻塞控制。

3.3优缺点
  • 速度快
  • 易丢包
3.4应用场景

要求通信速度高

  • 域名转换：DNS协议
  • 文件传输：FTP协议
  • 网络管理：SNMP协议
  • 远程文件服务器：NFS协议
3.5报文段格式

UDP协议的报文段共有首部字段（8字节）和数据字段

5.Socket

5.1简介

Socket并不是一种网络协议，而是一套封装了TCP/IP协议族的API，属于传输层。开发者使用Socket才可以在Android平台上进行基于TCP/IP协议的开发。开发者调用Socket组织数据，使其符合相关的协议，即可进行通信
